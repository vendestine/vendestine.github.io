Linux系统编程入门

Linux开发环境搭建

  1. VMware虚拟机安装Ubuntu22.04
  • 安装VMware Tools方便操作
  1. Xshell，Xftp 远程连接

  • 这里我们用SSH连接，所以远程系统里需要配置SSH
    sudo apt install openssh-server

  • 查看ip地址
    sudo apt install net-tools
    ifconfig

  • 然后配置Xshell， xftp远程SSH连接

  1. 远程编辑器VScode配置
  • 安装插件 Remote Development
  • 远程资源管理器 选择 SSH Targets 然后config 修改 host（随意取），hostname， user信息
  1. 配置免密登录

  • 本机拿到SSH公钥，如果没有以下语句生成
    ssh-keygen -t rsa 生成密钥

  • linux系统里.ssh目录下新建authorized_keys文件。然后将本机公钥copy过去


GCC

  • GCC 原名GNU C语言编译器（GNU C Compiler)，现在是GCC (GNU Compiler Collection, GNU编译器套件)，GNU 编译器套件包括 C、C++、Objective-C、Java、Ada 和 Go 语言，也包括了这些语言的库（如 libstdc++，libgcj等）

  • 安装 建议直接安装buildtools来安装gcc g++ //版本 > 4.8.5 （支持C++11）

  • 查看版本 gcc/g++ --version/-v

  • GCC工作流程
    源代码（.h .c .cpp) ->|预处理器| ->预处理后的源代码（.i) -> |编译器| -> 汇编代码（.s) -> |汇编器| -> 目标代码（.o) -> |链接器| -> 可执行程序(.exe .out)

其实就是经过四个步骤 预处理->编译->汇编->链接

  • gcc编译参数


常用的是 -c 汇编后停止 -g 支持调试

  • gcc无法链接，g++是编译+链接，其中编译实际是调用了gcc

静态库

库文件是计算机上的一类文件，可以简单的把库文件看成一种代码仓库，它提供给使用者一些可以直接拿来用的变量、函数或类
库是特殊的一种程序，编写库的程序和编写一般的程序区别不大，只是库不能单独运行。

  1. 库文件有两种，静态库和动态库（共享库），区别是：静态库在程序的链接阶段被复制到了程序中；动态库在链接阶段没有被复制到程序中，而是程序在运行时由系统动态加载到内存中供程序调用。

  2. 库的好处：1.代码保密 2.方便部署和分发

  • 静态库的命名规则
    Linux: libxxx.a //xxx是库饿名字 lib和a是固定的前缀和后缀
    windows：libxxx.lib

  • 静态库的制作

    1. gcc 得到 .o文件
    2. ar rcs libxxx.a xxx.o xxx.o //r 插入备存文件 c 建立备存文件 s索引

  • 静态库的使用
    一般工程项目我们结构如下
    –include
    –head.h
    –lib
    –libxxx.a
    –main
    –src
    –add.c
    –sub.c
    –…
    –…

然后这里示例一下如何制作和使用静态库

  1. src目录下 gcc -c 生成.o文件
    gcc -c xxx.c xxx.c -I ../include/

  2. 生成静态库
    ar rcs libxxxx.a xxx.o xxx.o

  3. 移动库到lib文件夹
    mv libxxx.a ../lib/

  4. 编译main.c
    ``gcc main.c -o app -I ../include -L ./lib/ -l xxx(库的名字）`
